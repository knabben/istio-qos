# Implementation Plan: Kind Development Environment Setup

**Branch**: `001-kind-istio-setup` | **Date**: 2026-04-26 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-kind-istio-setup/spec.md`

## Summary

Create a reproducible local development environment for `mesh-priority-controller` using
a kind cluster wired to a local container registry (port 5000), with Istio installed via
`istioctl` using the `demo` profile. Two idempotent shell scripts in `rec/` drive the
setup sequence; a `README.md` documents the full procedure for new contributors.

## Technical Context

**Language/Version**: Bash (POSIX-compatible, targeting bash 4+)
**Primary Dependencies**: kind, kubectl, istioctl, Docker
**Storage**: N/A — no persistent application data; cluster state lives in Docker
**Testing**: Manual verification per `quickstart.md`; idempotency verified by re-running scripts
**Target Platform**: Linux and macOS (Windows out of scope for v1)
**Project Type**: Infrastructure / developer tooling (shell scripts + documentation)
**Performance Goals**: Full setup completes in under 15 minutes on a standard developer machine
**Constraints**: Scripts must be idempotent; non-zero exit on any failure; no silent errors
**Scale/Scope**: Single-node kind cluster; local development only; no production use

## Phase 0: Research Findings

See [research.md](research.md) for full decision log. Summary:

| Question | Decision |
|----------|----------|
| Registry config | containerd mirror `localhost:5000 → http://kind-registry:5000` |
| Registry port | 5000 (kind convention) |
| Istio install method | `istioctl install` (Operator deprecated since 1.23) |
| Istio profile | `demo` (includes DestinationRule, VirtualService, sidecar injection) |
| Istio version | Pinned via `ISTIO_VERSION` var at script top |
| Idempotency | Pre-check each resource before creation; skip if exists |
| Script structure | Separate `bootstrap.sh` + `install-istio.sh` + `teardown.sh` |

See [contracts/scripts.md](contracts/scripts.md). Each script:
- Accepts overrides via env vars (`CLUSTER_NAME`, `REGISTRY_PORT`, `ISTIO_VERSION`, etc.)

- Exits non-zero on any failure
- Guarantees idempotency

## Implementation Notes

- `rec/bootstrap.sh` MUST create the registry before the cluster so the containerd patch
  can reference an already-running registry container name.
- The kind cluster config YAML is generated inline in `bootstrap.sh` using a here-doc.
- `rec/install-istio.sh` MUST validate that `istioctl` is in `$PATH` and print a clear
  download instruction if absent (do not auto-download without user consent).
- `README.md` MUST include the pinned Istio version number so readers know which version
  to install before running `rec/install-istio.sh`.
- All scripts MUST be executable (`chmod +x`) and include a `#!/usr/bin/env bash` shebang.
